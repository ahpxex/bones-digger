## 【开场 · 30 秒】

各位评委老师好。

这里是『骨鉴』—— **一个让一部手机、一张照片，就能做动物骨骼种属鉴定的 AI 工具**。

场景先说清楚：中国每年田野考古出土数以万计的动物骨骼碎片，但基层考古队普遍缺乏专家，超过九成被笼统记为"兽骨"。关键的物种信息就此丢失。我们想做的是**把考古专家的鉴定能力，用 AI 封装到每一位田野研究员的手机里**。

## 【问题与价值 · 1 分钟】

我们的需求来自中国社会科学院考古研究所的一线老师。原话是——"挖出来之后就想知道这是什么种属的骨骼，最好拍个照就能告诉我。"

这个需求看似简单，但难点有三层：
- 第一，骨骼鉴定是**细粒度分类**——马的距骨和牛的距骨区别只在"滑车是一个还是两个"这种细节；
- 第二，田野照片**不干净**——有泥土、手指、比例尺、风化痕迹，识别前必须分离骨骼主体；
- 第三，AI 不能胡说——考古数据一旦入档就是长期参考，**鉴定结果必须可溯源、可审计**。

传统做法要么训练一个 YOLO 黑盒给标签，要么拿 GPT-4V 自由问答。前者不可解释，后者可能编造。我们认为**可解释性比准确率更重要**——这是骨鉴区别于其它 AI 鉴定的第一个核心判断。

## 【解决方案架构 · 2 分钟】

骨鉴是一个 **Next.js 16 + React 19 + TypeScript 全栈 Web 应用**。零 Python 运行时，所有 AI 能力通过 HTTP API 组合起来。

核心流程五步：

**第一步**，用户用手机或桌面上传一张骨骼照片。

**第二步**，后端用 **SAM 3.1 Object Multiplex** 去除泥土、手、比例尺等干扰，提取骨骼主体。SAM 3.1 是 Meta 上个月——2026 年 3 月 27 号——刚开源的最新代分割模型，支持文本 prompt，我们用 `"bone, skeleton"` 一句话就能激活。

**第三步**，我们的 **RAG 引擎**工作。我们把 PRD 第 2.4 节里的专家鉴定特征——7 个物种 × 18 个骨位，73 条权威要点——通过阿里 **Qwen3-Embedding**（text-embedding-v4，Qwen3 家族最新代）向量化入库。用户上传时，把图像描述和用户补充信息做检索，选出最相关的 Top-K 条专家特征注入大模型 prompt。

**第四步，这是整个系统的心脏——双通道 Qwen3.5 推理**：
- **实时通道**用 Qwen3.5-Flash，2 月发布的原生多模态模型。低延迟路径，大部分样本 3 秒出结果。
- **精推通道**用 Qwen3.5-Plus 带 Thinking 模式，输出可审计的思维链。**只有当实时通道置信度低于 0.72 时才自动触发**，控制成本和延迟。两个通道判定一致时，置信度融合加成；不一致时，信任精推结果并保留原始冲突记录。

**第五步**，结果同步跑 **SAM 3D**——Meta 3 月另一款新品，单张照片直接重建为 GLB 格式的 3D 网格。田野研究员**不需要环拍**，同一张照片既出鉴定，又出数字标本。

整条链路全部用最新代模型，全部走托管 API。为什么不训练自己的模型？因为在 RAG + 充分专家知识库的前提下，**预训练 VLM + 检索增强**在小数据集上已经比从头训细粒度分类器更稳、更快、更可解释。这是我们的第二个核心判断。

*（滑到画廊）* 这里是 16 件从真实 300 张标本里抽样的 demo，评委可以点任意一张直接鉴定。我点**第 11 件**——

*（点击，等待 3-10 秒）* 好，结果出来了。**黄牛 · 尺骨 · 置信度 75.0%**，拉丁名 Bos taurus · Ulna。

我们看**三联画**：左边原片，中间 SAM 分割出的骨骼主体（青铜金边框），右边热力图——红色光斑标注了 Qwen3.5 视觉模型关注的两个关键区域：**尺桡关系**和**结节形态**，和右侧证据逐条对应。

往下滑——**六维度评分雷达图**：整体形态、关键结构、截面断面、表面纹理、表面痕迹、尺寸比例。评委可以一眼看到模型在哪些维度最有信心。

**证据链**：第一条，尺桡关系，权重 42%。图像观察：**"可见独立分离的两根长骨主体，间隙明显"**。专家特征（这是 RAG 引擎从知识库里取出的原文）：**"黄牛·尺骨：与桡骨等长，之间有缝隙"**。二者直接对照——图像观察永远要映射到一条可引用的专家条目，AI 不能脱离知识库编造。

第二条证据讲结节形态，同样对照。

**思维链**：大模型把从分割到判定的完整推理写出来。这里可以看到模型明确排除了"马类尺桡愈合"这个竞争假设。

最后是**关联知识卡**，展示同一骨位在其它几个物种的特征对照，方便研究员做二次核验。

**一键导出 PDF 鉴定报告**——文物鉴定证书样式，右下角朱印"骨鉴"，可以直接打印入档。

*（点 3D 标本段）* 这里是 SAM 3D 对同一张照片的三维重建。鼠标可以拖动旋转，关键特征区域直接标注在网格上。

## 【技术对比 · 1 分钟】

我们系统的技术底座**全部是赛事提交前 2 个月内发布的最新代模型**：
- 2026-02-16 · Qwen3.5
- 2026-03-27 · SAM 3.1 Object Multiplex
- 2026-03 · SAM 3D Objects

和同类工具对比：
- **传统 YOLO 分类器**：输出标签 + 概率，**不可解释**。
- **直接 GPT-4V 提问**：可以聊，但**没有知识库约束，可能生成不实特征**。
- **骨鉴**：判定 + 证据 + 权重 + 推理链 + 原文引用，**全链路可溯源**。

可解释性不是装饰——田野考古数据一旦入档就是几十年的参考依据，必须能回答"AI 为什么这样判断"。

## 【数据与验证 · 1 分钟】

当前：7 物种 × 18 骨位 × 73 条专家特征 × 300 张真实标本（中科院考古所提供，前半马后半牛）。我们已经把其中 16 张通过真 Qwen3.5 批量鉴定入库，/history 页可以看到全部 16 条真实输出。

不做大规模 benchmark 的原因：**我们不训模型**。在 RAG 范式下，benchmark 的意义是"知识库覆盖度 + VLM 零样本能力"的联合评估。下一阶段扩数据到 7 物种 × 500 张，再做系统性测试。

## 【风险与兜底 · 30 秒】

四个明确的失败模式和对应的兜底策略：

1. **VLM 置信度不足** → 精推通道自动复核
2. **图像超出 7 物种** → OOD 守护栏，返回"未知"标签和"建议送专家"横幅——不硬猜
3. **答辩现场网络挂** → Mock provider 本地确定性输出 + 预录 demo 视频
4. **知识库覆盖不足** → /admin 知识库维护界面，专家持续录入形成闭环

骨鉴不只是一个比赛作品。中国考古研究所说过，如果我们做得成熟，可以真正用到田野现场——把每年上万件的"兽骨"标签，一点点还原成有名有姓的历史证据。
